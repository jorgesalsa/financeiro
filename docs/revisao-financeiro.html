4. Baixa (Settlement)

Fluxo de Status do Lancamento na Baixa

OPENNenhum pgto
→
Baixa parcialSettlement criado
→
PARTIALpaidAmount < amount
→
Baixa finalCompleta valor
→
SETTLEDpaidAmount ≥ amount

Formulas da Baixa

totalJaPago = SOMA(settlements anteriores)
restante = entry.amount - totalJaPago
valorBaixa = amount + juros + multa - desconto

Validacao: amount ≤ restante + 0.01 (tolerancia float)
Status: SE totalPago ≥ amount - 0.01 ENTAO "SETTLED" SENAO "PARTIAL"
Saldo banco: SE category=PAYABLE ENTAO saldo -= valorBaixa SENAO saldo += valorBaixa

Campos Atualizados no OfficialEntry

CampoLogica de Atualizacao
statusSETTLED se totalPago >= amount, senao PARTIAL
paidDateData da baixa (so quando totalPago >= amount)
paidAmountSoma acumulada de todas as baixas
interestAmount+= juros desta baixa (acumulativo)
fineAmount+= multa desta baixa (acumulativo)
discountAmount+= desconto desta baixa (acumulativo)

Logica de Geracao de Parcelas

valorParcela = TRUNCAR(total / numParcelas, 2 casas)
ultimaParcela = total - (valorParcela * (numParcelas - 1))

Cron: /api/cron/recurring-entries (POST)

PassoAcao
1Busca regras: active=true E nextGenerationDate ≤ hoje
2Para cada regra, cria StagingEntry com status=AUTO_CLASSIFIED
3Calcula proxima data baseado na frequencia
4Se nextDate > endDate, desativa regra (active=false)

Inconsistencia Encontrada

#P3 Cron cria StagingEntry (passa pelo fluxo staging → oficial).
Service (recurring.ts) cria OfficialEntry diretamente (bypass staging).
O cron e o que roda em producao. O service e codigo morto.
BUG O cron nao passa dueDate nem competenceDate no StagingEntry criado (campos nao existem no schema ainda).

8. Deteccao de Atraso (Overdue Cron)

Cron: /api/cron/overdue-check (POST)

PassoAcao
1Busca entries: status IN (OPEN, PARTIAL), category IN (PAYABLE, RECEIVABLE), dueDate < hoje
2Para cada entry vencida, cria AuditLog com detectedOverdue=true
3Conta payables vs receivables vencidos
Problema #P4: O cron APENAS grava AuditLog. Nao cria Notification, nao altera status, nao envia email. O usuario so descobre que tem contas vencidas se entrar na tela de excecoes ou no dashboard.
